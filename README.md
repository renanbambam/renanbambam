### About / Sobre

**🇬🇧 EN —** I'm a fullstack developer with a software-engineering foundation: my focus is on **programming logic, architecture, system integration and strong engineering fundamentals** — not on any single language.

Since **2024 I've been at PRODEMGE**, working on a corporate govtech system used by real public-sector users, where I evolve domain rules, work with **Oracle SQL** and resolve production incidents within an Agile (Scrum/Kanban) team. My professional stack centers on **backend with Java / Spring Boot and Oracle**, plus frontends in **React, Next.js and Angular**.

Alongside that I have hands-on experience and applied study in **C# / .NET, Python, TypeScript and mobile development**, and I'm actively interested in **Salesforce** and native mobile with **Swift / Kotlin**. This foundation lets me enter different stacks with a real technical base rather than starting from zero.

I'm open to **junior fullstack positions beyond Java** — I'm not tied to one language, but to design quality, architecture and the fundamentals that make systems maintainable.

**🇧🇷 PT —** Sou desenvolvedor fullstack com base em **engenharia de software**: meu foco está em **lógica de programação, arquitetura, integração entre sistemas e fundamentos sólidos de engenharia** — não em uma linguagem específica.

Desde **2024 atuo na PRODEMGE**, em um sistema corporativo govtech com usuários reais do setor público, onde evoluo regras de negócio, trabalho com **Oracle SQL** e resolvo incidentes de produção em time Ágil (Scrum/Kanban). Minha stack profissional gira em torno de **backend com Java / Spring Boot e Oracle**, além de frontends em **React, Next.js e Angular**.

Em paralelo, tenho experiência prática e estudos aplicados em **C# / .NET, Python, TypeScript e desenvolvimento mobile**, e interesse ativo em **Salesforce** e mobile nativo com **Swift / Kotlin**. Essa fundação me permite entrar em diferentes stacks com base técnica real, sem começar do zero.

Estou aberto a **posições júnior em desenvolvimento fullstack além do Java** — não sou limitado a uma linguagem, e sim focado em qualidade de design, arquitetura e nos fundamentos que mantêm sistemas sustentáveis.

### Featured Projects / Projetos em destaque

**🧩 Finance Manager — a three-tier product / um produto em três camadas**

🇬🇧 A single financial-management domain implemented across backend, web and mobile, sharing one auth model and API contract.
🇧🇷 Um único domínio de gestão financeira implementado em backend, web e mobile, compartilhando o mesmo modelo de autenticação e contrato de API.

| Tier / Camada | Repo | Stack | Highlights / Destaques |
|------|------|-------|-----------|
| **API** | [`finance-api-quarkus`](https://github.com/renanbambam/finance-api-quarkus) | Java 21 · Quarkus · MongoDB | JWT + refresh-token rotation, RBAC, layered architecture |
| **Web** | [`finance-manager-web`](https://github.com/renanbambam/finance-manager-web) | Angular 16 | JWT interceptor, route guards, Chart.js dashboards |
| **Mobile** | [`finance-manager-mobile`](https://github.com/renanbambam/finance-manager-mobile) | React Native · Expo | Offline-first SQLite, Redux Toolkit, charts |

**🛡️ Organization Management API**

[`org-management-api`](https://github.com/renanbambam/org-management-api) — NestJS · MongoDB · TypeScript

🇬🇧 Hierarchical **Company → Branch → User** model with a five-level RBAC scheme, JWT access + refresh strategies, and a deny-by-default security pipeline (helmet, rate limiting, strict validation) documented via OpenAPI. A second take on the same domain as `finance-api-quarkus`, in a different ecosystem.
🇧🇷 Modelo hierárquico **Empresa → Filial → Usuário** com RBAC de cinco níveis, estratégias JWT de acesso + refresh e um pipeline de segurança *deny-by-default* (helmet, rate limiting, validação estrita) documentado via OpenAPI. Uma segunda abordagem do mesmo domínio de `finance-api-quarkus`, em outro ecossistema.
