## About

I'm a fullstack software engineer with a strong foundation in engineering fundamentals — my value comes from architecture, programming logic, integration and maintainability, not from any single technology.

Since **2024 I've been a Backend Engineer at PRODEMGE**, working in a real production environment with real public-sector users. My day-to-day centers on **backend with Java / Spring Boot**, **Oracle SQL** and enterprise systems, where I evolve domain rules and resolve production incidents. Beyond that, I have hands-on experience and applied study in **C# / .NET, Python, TypeScript and mobile development**, with an active interest in **Salesforce, Swift and Kotlin**. Because my focus is on engineering principles and problem solving rather than a specific language, I can move into different stacks with a real technical base.

## Featured Projects

### 🧩 Finance Manager Ecosystem

A single financial-management product implemented across three layers — API, web and mobile — sharing one authentication model and API contract.

**API — [`finance-api-quarkus`](https://github.com/renanbambam/finance-api-quarkus)**
`Java 21 · Quarkus · MongoDB`
- JWT authentication with refresh-token rotation
- Role-based access control (RBAC)
- Layered architecture (model → repository → resource)
- Domain modeling for enterprises, expenses, income and invoices

**Web — [`finance-manager-web`](https://github.com/renanbambam/finance-manager-web)**
`Angular 16 · TypeScript · Chart.js`
- JWT interceptor and route guards
- Dashboards with Chart.js
- Reactive forms and typed services

**Mobile — [`finance-manager-mobile`](https://github.com/renanbambam/finance-manager-mobile)**
`React Native · Expo · Redux Toolkit`
- Offline-first persistence with SQLite
- Centralized state with Redux Toolkit
- Charts and shared API contract with the backend

### 🛡️ Organization Management API

**[`org-management-api`](https://github.com/renanbambam/org-management-api)**
`NestJS · MongoDB · TypeScript`

A second take on the same domain as `finance-api-quarkus`, in a different ecosystem.
- Hierarchical **Company → Branch → User** model
- Five-level RBAC scheme
- JWT access + refresh token strategies
- Deny-by-default security pipeline (helmet, rate limiting, strict validation)
- OpenAPI documentation

### 📈 Trader Assist Showcase

**[`trader-assist-showcase`](https://github.com/renanbambam/trader-assist-showcase)**
`Architecture · Documentation`

Documentation-only showcase of a real-time trading assistant, presenting its system design without exposing proprietary logic.
- Event-driven communication between services
- WebSocket realtime updates
- Context persistence system
- Dockerized development environment
- CI/CD with GitHub Actions
